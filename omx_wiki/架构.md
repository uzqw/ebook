# 架构

```text
Browser
  -> Vue Router 页面
  -> PocketBase JS SDK: auth/books/bookmarks/notes/reading_records
  -> <img> 请求 /api/books/{id}/pages/{page}/image?token=...

Extended PocketBase (Go)
  -> REST API / auth / SQLite / file storage
  -> OnRecordAfterCreateSuccess("books") 触发解析
  -> PDF: go-pdfium WebAssembly 解析文本/目录并渲染 PNG
  -> EPUB/MOBI: go-fitz/MuPDF 解析文本/目录并渲染 PNG
```

## 为什么使用 PocketBase

PocketBase 负责认证、SQLite、文件存储、CRUD API 和管理后台。文档解析/渲染属于 PocketBase 原生能力之外，因此用 Go 扩展实现。

## 数据流

1. 前端上传书籍到 `books.file`，同时写入 `user/title/parse_status=pending/current_page=1`。
2. Go hook 将书籍标记为 `processing`。
3. 后端按扩展名选择解析器：
   - PDF：`go-pdfium` 读取页数、PDF outline、每页文本和页面尺寸。
   - EPUB：先在临时 EPUB 副本中注入 CJK 字体，再用 `go-fitz` 解析页数和每页文本；目录优先读取 EPUB3 `nav.xhtml`，没有可用 nav 时回退到 `toc.ncx`，并通过页面文本定位真实 MuPDF 页码。
   - MOBI：用 `go-fitz` 解析页数和每页文本，目录 best effort。
4. 解析完成后写入：
   - `books.page_count`
   - `books.toc`
   - `book_pages.page_number/text/width/height`
   - `books.parse_status=completed`
5. 阅读页请求图片路由，后端校验 token 和书籍归属后渲染 PNG；前端用 `books.toc` 提供目录跳页，无目录时回退为页面索引。

## 解析器边界

- PDF 渲染走 `go-pdfium`，支持并发池。
- EPUB/MOBI 渲染走 `go-fitz`，受 native MuPDF 约束，后端用 `fitzMu` 串行化所有 fitz 入口。
- EPUB 的 CJK 字体注入必须同时用于解析和渲染，确保“本页文本”和图片页码一致。
