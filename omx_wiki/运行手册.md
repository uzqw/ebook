# 运行手册

## 初始化

```bash
cp .env.example .env
make setup
```

## 启动

终端 1：

```bash
make backend
```

终端 2：

```bash
make bootstrap
```

终端 3：

```bash
make dev-frontend
```

## 验证

```bash
npm run check
npm run build
cd backend && go test ./...
```

手动链路：用 `demo@reader.local / ebook-reader-user-123` 登录，上传 PDF 或 EPUB，等待解析完成，进入阅读页查看页面图片、目录跳转和“本页文本”。

## 环境变量

```bash
PDF_RENDER_DPI=220
EPUB_RENDER_FONT=/usr/share/fonts/droid/DroidSansFallback.ttf
```

- `PDF_RENDER_DPI`：PDF/EPUB 页面渲染清晰度默认使用后端 `renderDPI()`，可设 72-360；越高越慢且图片越大。
- `EPUB_RENDER_FONT`：EPUB 渲染用 CJK 字体路径。留空时后端会尝试 DroidSansFallback 和 Noto CJK 常见路径。部署环境如果 EPUB 图片缺中文，优先检查该变量和字体文件。

## 目录解析

- PDF：读取 PDF outline/bookmarks 写入 `books.toc`。
- EPUB：优先读取 EPUB3 `nav.xhtml`；如果没有可用 nav，再回退到 `toc.ncx`。NCX 只有 `Start` 的样本通常是兼容占位，不应直接依赖 `go-fitz` 的 `ToC()` 结果。
- EPUB 目录页码不能只用 OPF spine 顺序，因为 MuPDF 会重分页；后端会用目录标题在解析后的页面文本中定位真实页码，并跳过目录页本身。
- 如果目录为空，前端会回退为页面索引。
- 上传后如果解析还在 `pending` / `processing`，前端会自动等待或轮询；不要把这个阶段误判为目录解析失败。

## EPUB 排障

### `warning: unknown epub version: 3.0`

MuPDF 的非致命警告，不代表解析失败。只要 `parse_status=completed` 且页面能打开，可忽略。

### 图片有英文/线条但缺中文

通常是 CJK 字体不可用。检查：

```bash
ls -l "$EPUB_RENDER_FONT"
fc-list :lang=zh | head
```

解决：安装 Droid/Noto CJK 字体，或在 `.env` 设置 `EPUB_RENDER_FONT` 后重启后端。

### 图片和“本页文本”对不上

说明解析文本和渲染图片使用的 EPUB 分页不一致。当前约定是：EPUB 解析和渲染都必须先调用 `injectEPUBRenderFont`，使用同一份注入字体后的临时 EPUB。改动相关逻辑时要同时验证图片和 `book_pages.text`。

### 旧记录卡在 `processing`

通常是旧版本 native 崩溃遗留状态。可删除后重新上传，或补跑解析脚本/临时工具重写 `book_pages` 与 `books.toc`。
